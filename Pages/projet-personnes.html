Description du projet

Ce projet consiste en le développement d'une application C# de gestion de personnes connectée à une base de données SQL Server. L'utilisateur peut saisir les informations d'une nouvelle personne via un formulaire, les enregistrer en base, consulter la liste complète des personnes enregistrées et supprimer une entrée.

L'application suit le principe CRUD (Create, Read, Delete) et a été développée en WinForms sous Visual Studio. Elle m'a permis de mettre en pratique la connexion à une base de données depuis une application C# desktop, ainsi que l'exécution de requêtes SQL paramétrées.

Fonctionnalités

  • Formulaire d'ajout d'une personne (nom, prénom, date de naissance, email)
  • Enregistrement des données en base de données SQL Server
  • Affichage de la liste complète des personnes dans un tableau
  • Sélection d'une personne dans la liste pour voir le détail de ses informations
  • Suppression d'une personne avec confirmation avant suppression
  • Actualisation automatique de la liste après chaque ajout ou suppression
  • Validation des champs avant insertion (champs obligatoires, format email)

Difficultés rencontrées

La première difficulté a été d'établir correctement la connexion à SQL Server depuis C# via une chaîne de connexion. Comprendre le fonctionnement de SqlConnection, SqlCommand et SqlDataReader a nécessité de la pratique avant d'être à l'aise avec ces outils.

L'utilisation de requêtes paramétrées pour éviter les injections SQL était une notion nouvelle pour moi. Il était tentant d'utiliser des concaténations de chaînes directement dans les requêtes, mais j'ai appris pourquoi c'était une mauvaise pratique et comment utiliser les paramètres correctement.

Enfin, synchroniser l'affichage du tableau avec les données réelles de la base après chaque opération (ajout, suppression) sans recharger toute l'interface a demandé une réflexion sur la gestion de l'état de l'application.
